# RoadWorld ⇄ Pixel Foundry Integration

**RoadWorld role:** consumes Pixel Foundry scenes as walkable maps / worlds.
**Pixel Foundry role:** the scene production pipeline that feeds RoadWorld.

Canon doc: `docs/products/blackboard/PIXEL_FOUNDRY_CANON.md`

---

## What flows from Pixel Foundry → RoadWorld

When a Pixel Foundry scene is sent to RoadWorld, the export package contains:

- **tile map** (terrain layer, 40 × 25 default)
- **sprite sheet** (32 × 32 native sprites used by the scene)
- **object list** (props, NPCs, collectibles with positions)
- **collision map** (per-tile `blocked` flags)
- **interaction zones** (which tiles trigger dialog / warp / collect)
- **scene metadata** (`title`, `category`, `theme`, `era`, `weather`)
- **spawn point** (`{x, y}` where the player drops in)
- **room title** (HUD label)
- **environment status** (e.g., "noon · clear · 72°F")
- **route labels** (off-edge connections to neighbor scenes)
- **public-safe scene package** (everything passes the safety checklist)

## Cross-scene navigation

A scene can declare neighbors via `metadata.neighbors`:

```json
{
  "neighbors": {
    "north": "scene:zen-garden",
    "south": "scene:bus-stop-br-07",
    "east":  "scene:beach",
    "west":  null
  }
}
```

When the player walks off an edge, RoadWorld reads the neighbor link and warps to the next scene.

## RoadWorld UX

- User opens any Pixel Foundry scene → RoadWorld renders it walkable
- Standard controls: `WASD` / `↑↓←→` walk, `E` interact, `Shift` sprint
- HUD: scene title (top-left), status (top-right), interaction toast (bottom-center)
- Optional minimap built from the foundry scene's `map[][]` array

## Earth tile-slot host (proposed)

Per the user's vision: a zoomable Earth where each square at deep zoom maps to a Pixel Foundry scene slot. Empty squares display "claim this square → opens World Builder pre-pinned to that lat/lon."

When a square is claimed, its scene HTML lives at `roadworld.blackroad.io/<lat>,<lon>/` and seamlessly enters from neighboring squares.
